# <center> Arquitetura e Desenho de software

<div align="justify">
A arquitetura e o desenho de software é aquilo que liga os artefatos com um nível de abstração bem alto com as características em um nível mais baixo de abstração, nível de software e hardware. Ou seja, vai passando de engenharia de requisitos para o desenho de software.

Trata-se de uma atividade do núcleo (core) técnico da Engenharia de Software, sendo aplicada independentemente do modelo de processo de software usado. Basicamente, os elementos do modelo de análise (ex.casos de uso) fornecem insumos para a criação dos modelos de especificação do projeto de software em desenvolvimento. 

Lapidando o modelo de projeto, vamos descendo o nível de abstração até chegar no código. Por isso que requisitos é a primeira coisa, porque ela vem lá do cliente, a visão dele, a visão geral e bem abstrata do que se deseja desenvolver.

|<center>Engenharia de Requisitos | <center>Desenho de software|
| :-- | :-- |
| • Modelagem do problema (entender)<br>• Qual é o problema?<br>• Como ele ocorre?<br>• Domínio do problema<br> • **Mundo real** | • Modelagem da solução (criar)<br>• Qual é a solução? <br>• Como ela ocorre?<br>• Domínio da Solução<br>• **Mundo Computacional**

Resumindo, o desenho de software define o quê é a solução (projeto arquitetônico – estratégico) e define como é a solução (projetos lógico e detalhado – tático e operacional), ou seja, como cada componente deve ser implementado. O objetivo disso tudo é produzir um projeto de solução, o qual deve ser implementado nas especificações de hardware e software definidas.

Os artefatos e pré-rastreabilidade utilizados pra desenho são criados bem no começo, mesmo quando não se sabe muito bem sobre o projeto. Eles ajudam a equipe a decidir quais são as melhores tecnologias a serem usadas no projeto.
